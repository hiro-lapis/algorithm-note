#数列

## 問題文の把握

- 長さ N の値不明の数列 A が与えられる
- 長さ N-1 の整数列 B が与えらえれる
- Bi >= max(Ai, Ai+1) が成立する
- A の要素のトータルとして考えうる最大値を求めよ

- 制約
- 2 <= N <= 100
- 0 <= Bi <= 10^5

```
N
B1 B2 ,,, B(N-1)

case1
3 
2 5
=> 9

case2
2
3
=> 6

case3
6
0 153 10 10 23
=> 53
```


## ロジック

### 読解
問題文が持って回った感じの内容なので、意味を理解するところから始める

- Bi >= max(Ai, Ai+1) が成立する

↑について、数列 A = [10, 13, 3] だとしたら、
B1 >= 13 となる(max(10, 13) で、連続する数値要素のうち大きい方と比較することになるため)

- A の要素のトータルとして考えうる最大値を求めよ
最大値について case1 を例に考えてみると、、、

```
case1
3 
2 5
=> 9

数列A の長さは 3 で、Bの長さは 2 

数列B = [2, 5]

B1 は 2 なので、
2 >= max(2, 1)  
2 >= max(-10000, 1)  
```

みたいに、A の要素の値はさまざまありうる   
ただし、問題文に「要素の総和が最大となる値」とあるため、
B1 の時点での数列 A の値は `2 >= max(2, 2)` つまり `A = [2, 2]` となることが導き出せる  
さらに B2 について考えてみると、

```
B2 は 5 なので、
5 >= max(5, 5)  
5 >= max(-10000, 4)
```

これもB1 同様、様々考えられ、`5 > max(Ai, Ai+1)` の最大の組み合わせはmax(5, 5)  だが、
最初の B1 の時点で `A1 = 2, A2 = 2` であることは決定している  

よって、A3 の値だけが最大値として考慮する対象になる
よって、`A3 = 5` となる

制約 0 <= Bi <= 10^5 であることと、A の総和の最大値を求める性質から、  
実質制約 0 <= Ai <= 10^5 であることもわかる  

### AB 間の関係まとめ
A B 間の関係をざっくり図にすると、以下のようになる  

```
B [0, 153, 10, 10, 23]
   |\  |\  |\  |\  |\
A [0, 0, 10, 10, 10, 23]

↑の縦線で結ばれている値において、Bi > Ai +Ai+1 の関係が成り立つ
制約 0 <= Bi <= 10^5
```

これをコードに起こせば、自ずと回答が導き出せそう

数字の長さは最大 100 なので、fmt.Scan() で大丈夫
